## 📁 ディレクトリを作成する（mkdir）

### ⚙️ 仕組み

`mkdir` は 新しいディレクトリ（フォルダ）を作成するコマンド。

🗂 Linux では、作業内容ごとにディレクトリを分けて管理するのが基本。

### 🧪 例

```bash
mkdir test
```

```bash
mkdir work logs
```

### 💡 ポイント

- 📦 複数のディレクトリ名を指定するとまとめて作成できる

- ⚠️ 同じ名前のディレクトリが存在するとエラーになる

- 🧹 作業前にディレクトリを作るクセを付けると整理しやすい

### ⭐ よく使うオプション

```bash
mkdir -p work/app/logs
```

- 🔗 -p を付けると**途中のディレクトリが存在しなくても一気に作成できる**
- 🏢 実務でも頻出

---

## 📝 空ファイルを作成する（touch）

### ⚙️ 仕組み

`touch` は 空のファイルを作成するコマンド。

設定ファイルやメモ用ファイルをとりあえず用意したいときに使う。

### 🧪 例

```bash
touch memo.txt
```

```bash
touch a.txt b.txt
```

### 💡 ポイント

- ⏱ すでに存在するファイルを指定すると更新日時のみが変更される

- 📦 「ファイルの枠だけ作る」用途で便利

---

## 📂 ファイル・ディレクトリの一覧を確認する（ls）

### ⚙️ 仕組み

`ls` は ディレクトリ内のファイルやフォルダを一覧表示するコマンド。

作成したファイルやディレクトリが正しく存在しているかを確認するために使う。

### 🧪 例

```bash
ls
```

```bash
ls -l
```

### 💡 ポイント

- ✅ 作成後は ls で確認するクセを付ける

- 🔍 -l を付けると詳細情報（権限・サイズなど）が分かる

---

## 📄 ファイルの中身を表示する（cat）

### ⚙️ 仕組み

cat は ファイルの内容をそのまま表示するコマンド。

短いテキストファイルの確認に向いている。

### 🧪 例

```bash
cat memo.txt
```

### 💡 ポイント

- ⚡ ファイルの内容を一気に表示する

- ⚠️ 長いファイルには向かない

---

## 📖 ファイルをスクロール表示する（less）

### ⚙️ 仕組み

less は ファイルの中身をスクロールしながら表示できるコマンド。

ログファイルや設定ファイルなど、内容が長いファイルの確認によく使われる。

### 🧪 例

```bash
less /etc/passwd
```

### 🎮 操作方法

- ⬆ ⬇ ：スクロール

- q ：終了

### 💡 ポイント

- 👍 cat より実務向き

- 🔁 Linux 操作では頻出コマンド

### 🔄 よくある作業の流れ

```bash
mkdir work
cd work
touch memo.txt
ls
cat memo.txt
```

- 🧭「作る → 移動 → 確認」が基本の流れ。

### ✅ まとめ

| コマンド   | 内容               |
| ---------- | ------------------ |
| 📁 `mkdir` | ディレクトリを作成 |
| 📝 `touch` | 空ファイルを作成   |
| 📂 `ls`    | 一覧を表示         |
| 📄 `cat`   | 内容を表示         |
| 📖 `less`  | スクロール表示     |
