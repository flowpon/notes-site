## 📄 ログ確認とは？

Linux サーバーでは  
**「何が起きているか」** を知るために **ログ確認** が必須。

- サービスが起動しない
- エラーが出ている
- 設定変更後の動作確認

👉 こうした場面で **まずログを見る** のが基本。

---

## 📝 ログファイルの場所（例）

多くのログは `/var/log` 配下にある。

```bash
ls /var/log
```

```text
syslog
messages
auth.log
```

※ 表示内容・ファイル名は **環境によって異なる**

---

## 👀 ファイルの中身を表示する（cat）

### ⚙️ 仕組み

`cat` は **ファイルの内容をそのまま表示**するコマンド。

### 🧪 例

```bash
cat /var/log/syslog
```

### 💡 ポイント

- ファイルが小さい場合に向いている

- ログが多いと **一気に流れて読めない** ことがある

👉 **軽い確認用**

---

## 📖 ゆっくり閲覧する（less）

### ⚙️ 仕組み

`less` は **スクロールしながらファイルを閲覧**できるコマンド。

### 🧪 例

```bash
less /var/log/syslog
```

### ⌨️ よく使う操作

| 操作      | 内容       |
| --------- | ---------- |
| `⬆ / ⬇`   | スクロール |
| `/文字列` | 検索       |
| `q`       | 終了       |

### 💡 ポイント

- 実務で最も使うログ閲覧コマンド

- 大きなログでも安全に確認できる

👉 **迷ったら less**

---

## ⏱ 最新のログを確認する（tail）

### ⚙️ 仕組み

`tail` は **ファイルの末尾（最新部分）を表示**するコマンド。

### 🧪 例1

```bash
tail /var/log/syslog
```

```bash
tail -n 50 /var/log/syslog
```

- -n 50：最新50行を表示

### 🧪 例2

```bash
tail -f /var/log/syslog
```

👉 ログが追加されるたびに画面が更新される。

### 💡 使用シーン

- サービス再起動時

- 設定変更直後

- エラー発生の瞬間を見たいとき

終了は `Ctrl + C`

## 🧠 実務でよくあるログ確認の流れ

```bash
ssh user@192.0.2.10
cd /var/log
ls
less syslog
tail -n 30 syslog
exit
```

👉 **接続 → 場所確認 → 閲覧 → 最新確認**

---

## ⚠️ ログ確認の注意点

- 本番サーバーでは **削除しない**

- 編集もしない（基本は閲覧のみ）

- 個人情報や認証情報が含まれることがある

---

## ✅ まとめ

| コマンド     | 役割             |
| ------------ | ---------------- |
| 📄 `cat`     | 中身を一気に表示 |
| 📖 `less`    | スクロール閲覧   |
| ⏱ `tail`     | 最新ログを確認   |
| 🔄 `tail -f` | リアルタイム監視 |

👉 **トラブル対応は「まずログを見る」から始まる**
